Introduction

Hypertext Markup Language (HTML) is the standard language for documents which are displayed on web browsers. HTML defines content and structure that will be displayed to the end user.

HTML is often associated with CSS, or Cascading Style Sheets and Javascript.

  • The current iteration of HTML is HTML-5.
  • HTML was initially released in 1993, 31 years ago.
What You Should Already Know

This guide is for entry-level web development students. Working knowledge of the following are helpful:

  • Basic knowledge of HTML structure
  • Functions of HTML elements

If you are unfamiliar with anything mentioned in this document, please refer to freeCodeCamp.com

Boiler Plate

Getting Started

To get started with writing HTML, open Notepad, or VSCode and name the file index.html

A basic boiler plate will include the following elements

!DOCTYPE html

  • This will tell the browser the document will use HTML-5, the most current version.

html lang="en"

  • The html element will contain everything rendered in HTML and tell the browser the default language is English

meta charset="UTF-8"

  • UTF-8 is the standard encoding character set used by HTML
  • UTF-8 will tell the browser to use UTF-8 to translate binary to characters, which will be displayed as text/numbers on a website.

title

  • title will display the title of the page in the tab and is used for screen reader accessibility purposes

meta name="viewport" content="width=device-width, initial-scale=1.0"

  • Tells the browser to display the page as the device's width with a scale of 1.0 to start.
  • Viewport is important for scaling and sizing in CSS styling. Elements can be set to percent of the viewport

link rel="stylesheet" href="styles.css"

  • Tells the browser to used the linked CSS stylesheet to apply user-made styles to the web page.
  • link can also link custom/non-standard font-families.
HTML Structure

This section will cover the basic hierarchy of HTML elements

html contains all HMTL elements on a webpage

head contains meta, links, title elements.

main differentiates content on page from navigation links or ads

nav used to contain all the navigation links on a webpage

body contains the main content of a page between the header and footer

div generic container element for text, images, links

section, article more specific container for text segments on a webpage

p paragraph element for text within a div, section, article

img container element for an image

  • takes src for image link
  • self-containing element. Does not need a closing tag

a stands for anchor. Used to contain a hyperlink which can be wrapped around text or an img element

input used in form creation and buttons. Usually nested in a div or label element

Publishing website using GitHub

There are many ways to publish your projects and take your websites live. For a student, GitHub can be a great free or low-cost option to publish your work and build a portfolio. Below are the steps to publish your work with GitHub.

  1. Create a Github account
  2. Download GitHub Desktop and sign in to your GitHub profile
  3. Create a new repository with CTRL+N, or going to the file tab.
  4. Pick a file location and add a name and description. Click initialize repository with a read.me. Click create repository
  5. Click Publish Repository
  6. Click View on GitHub
  7. Go back to your folder for the website and add all your files to the newly created folder containing the read.me for the website
  8. Give the first summary a description of initial-commit. Subsequent edits should have a relevant description
  9. On the GitHub website, go to settings, scroll to the bottom and change the repository to public
  10. Go to GitHub Pages under settings and change branch from none to main
  11. In a few minutes, a link should be available for you to view your website
